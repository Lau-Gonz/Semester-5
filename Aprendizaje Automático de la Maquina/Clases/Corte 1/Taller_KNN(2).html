En cuanto a los tipos de variables, se pueden identificar dos tipos: variables categricas y variables numricas. Las variables categricas son aquellas que describen caractersticas que no se pueden medir de forma continua, como el sexo, la clase del billete y el puerto de embarque. Por otro lado, las variables numricas son aquellas que describen caractersticas que se pueden medir de forma continua, como la edad, la tarifa del billete y el nmero de hermanos/cnyuges a bordo.

Los valores de las variables categricas se caracterizan por ser discretos y limitados a un conjunto finito de opciones, mientras que los valores de las variables numricas son continuos y pueden tomar cualquier valor dentro de un rango especfico. Los tipos de datos que se usan son principalmente enteros y flotantes para las variables numricas, y cadenas de texto para las variables categricas.

Segn nuestra intuicin, las variables ms relevantes para predecir la columna "Survived" seran las siguientes:

Sexo: Se sabe que durante el naufragio del Titanic, se dio prioridad a las mujeres y los nios para el acceso a los botes salvavidas. Por lo tanto, es probable que el sexo de un pasajero tenga una fuerte relacin con su supervivencia.

Clase del billete: Se sabe que las personas en las clases ms altas tenan ms probabilidades de sobrevivir, ya que tenan acceso a mejores alojamientos y a los botes salvavidas de manera prioritaria. Por lo tanto, es probable que la clase del billete tenga una relacin con la supervivencia.

Edad: Los nios y los ancianos tambin tuvieron prioridad para acceder a los botes salvavidas. Por lo tanto, la edad de un pasajero puede tener una relacin con su supervivencia.

ibSp: Esta variable indica la cantidad de hermanos/cnyuges a bordo del Titanic. Como mencionamos anteriormente, la posibilidad de sobrevivir puede estar relacionada con tener a un familiar cercano a bordo, por lo que esta variable podra ser relevante para predecir la supervivencia.

Parch: Esta variable indica la cantidad de padres/hijos a bordo del Titanic. Similar a la variable SibSp, puede haber una relacin entre tener familiares cercanos a bordo y la probabilidad de sobrevivir.

Embarked: Esta variable indica el puerto de embarque del pasajero. Si bien esto puede no estar directamente relacionado con la supervivencia, puede estar relacionado con otros factores que influyen en la supervivencia, como la clase del billete o la nacionalidad del pasajero.

Por otro lado, considero que las siguientes variables pueden ser menos relevantes para predecir la columna "Survived":

Nombre: Similar a la variable nmero de billete, esta variable es principalmente identificadora y no aporta informacin significativa sobre la supervivencia de un pasajero.

En resumen, considero que las variables ms relevantes para predecir la supervivencia de un pasajero en el naufragio del Titanic son el sexo, la clase del billete y la edad, mientras que las variables menos relevantes son el nmero de billete, el nombre y la cabina.

Convertimos a dummies las columnas categricas. Nos aseguramos de que las originales no permanezcan en la tabla. Se sugiere usar el parmetro drop_first = True si est trabajando con Getdummies. Por qu cree que es conveniente usar este parmetro?

El parmetro drop_first=True en el mtodo get_dummies se utiliza para eliminar una de las categoras de una variable categrica, que acta como la categora de referencia o base, para evitar la multicolinealidad.

Cuando se crean variables dummies para variables categricas con n categoras, se crean n-1 variables dummies. La categora de referencia o base se utiliza como punto de referencia para comparar las otras categoras. Por lo tanto, una categora no necesita tener su propia variable dummy.

Por ejemplo, en el caso de la variable "Embarked", existen tres posibles valores: "C", "Q" y "S". Si creamos variables dummy para estas tres categoras, se crearn dos variables dummy ("Embarked_Q" y "Embarked_S"). El valor de "Embarked_C" se puede inferir por la falta de valores en las otras dos variables dummy. Por lo tanto, podemos eliminar una de las variables dummy para evitar la multicolinealidad y reducir el nmero de caractersticas.

En resumen, el parmetro drop_first=True se utiliza para crear una representacin nica de las categoras de una variable categrica y evitar la multicolinealidad en el conjunto de datos, lo que puede mejorar la precisin del modelo de aprendizaje automtico.
